Day 09 - Docker Integration with Jenkins & GitLab
🔹 Ansible - Setup Docker Container for Nginx  [Download Image -> Run Container -> Open Port]
- name: Setup Docker Container for Nginx				-----> Think of it like writing: Today's work: Start Nginx Container
  hosts: all								-----> run for all servers
  become: yes								-----> become administrator
  tasks:
    - name: Pull Nginx image
      docker_image:							-----> Imagine Docker Hub is a toy shop, your computer doesn't have toy yet, so first go to the shop
	name: nginx							-----> which toy? Nginx
	source: pull							-----> download it from Docker Hub (Docker Hub -> Download image) 
    - name: Run Nginx container
      docker_container:							-----> you downloaded the toy now start playing
	name: nginx-container						-----> Container's name 
	image: nginx							-----> use the nginx image 
	state: started							-----> start it 
	published_ports:						-----> This is very important, Imagine visitors knows on 8080 (door), inside the container (home) Nginx listens to 80 	
	  - "8080:80"

🔹 Terraform - EC2 with Auto Scaling Group [Template -> Need more servers -> Create Automatically]
resource "aws_launch_configuration" "example" {				-----> Think of it a photocopy machine. It stores instructions like: 1. which AMI? 2. Which instance type? Whenever AWS needs a new EC2 it uses this template.
  name = "example_config"
  image_id = "ami-0abcd1234abcd1234"					-----> Operating System
  instance_type = "t3.medium"						-----> server size
}

resource "aws_autoscaling_group" "example" {				-----> The manager
  desired_capacity	= 2						-----> I want 2 servers normally
  max_size		= 3						-----> never go above 3
  min_size		= 1						-----> never go below 1
  vpc_zone_identifier	= ["subnet-12345678"]
  launch_configuration	= aws_launch_configuration.example.id
}

🔹Kubernetes - Horizontal Pod Autoscaler (HPA) [One Pod -> High CPU -> Create More Pods -> CPU Low -> Remove Extra Pods]
apiVersion: apps/v1
kind: Deployment							-----> Create Pods
metadata:
  name: vishwesh-deployment
spec:
  replicas: 1								-----> Start with one Pod
  selector: 
    matchLabels:
      app: vishwesh-app
  template:
    metadata:
      labels:
	app: vishwesh-app
    spec:
      containers:
	- name: nginx
	  image: nginx
	  ports:
	    - containerPort: 80

---

apiVersion: Autoscaling/v2
kind: HorizontalPodAutoscaler
metadata:
  name: vishwesh-hpa
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: vishwesh-deployment
  minReplicas: 1							-----> Minimum one Pod
  maxReplicas: 10							-----> Maximum one Pod
  targetCPUUtilizationPercentage: 50					-----> if CPU reaches to 50%, add more pods.

🔹Jenkins - Deploy to Kubernetes [Jenkins -> Run kubectl -> Deploy App]
pipelines {
   agent any
   stages {
	stage('Deploy to Kubernetes') {					-----> Deploy the application
	    steps {
		script {
		    sh 'kubecl apply -f deployment.yaml'		-----> Send deployment.yaml to Kubernetes.
		}
	    }
	}
    }
}

🔹GitLab CI/CD - Deploy to Kubernets [Exactly the same idea. Instead of Jenkins, GitLab runs: kubectl apply: Deploy to Kubernetes. Again, your notes say kubec1; it should be kubectl.]
stages:
  - deploy

deploy:
  stage: deploy
  script:
    - kubecl apply -f Kubernetes/deployment.yaml
