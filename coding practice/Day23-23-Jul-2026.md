🔹 Ansible - Deploy lambda Functions Using AWS CLI
- name: Deploy AWS Lambda Function                                                             # → Gives a name to the Ansible playbook.
  hosts: localhost                                                                             # → Runs the playbook on the Ansible control node.
  tasks:                                                                                       # → Starts the list of automation tasks.
    - name: Create Lambda function                                                             # → Names the task to create an AWS Lambda function.
      command: aws lambda create-function --function-name vishwesh-lambda --runtime python3.8 --role arn:aws:iam::123456789012:role/lambda-excecution-role --handler lambda_funtion.lambda_handler --zip-file fileb://lambda.zip      # command: aws lambda create-function ... → Creates an AWS Lambda function using the AWS CLI. [--function-name vishwesh-lambda → Sets the Lambda function name. [--runtime python3.8 → Uses Python 3.8 as the Lambda runtime.] [--role arn:aws:iam::123456789012:role/lambda-execution-role → Assigns the IAM execution role to the Lambda function.] [--handler lambda_function.lambda_handler → Specifies the Python handler function.] [--zip-file fileb://lambda.zip → Uploads the Lambda deployment package.]

🔹Terraform - Serverless API Gateway with Lambda Integration
resource "aws_api_gateway_rest_api" "vishwesh_api" {                                           # → Creates an API Gateway REST API.
  name          = "vishwesh-api"                                                               # → Sets the API Gateway name.
  description   = "vishwesh API"
}

resource "aws_lambda_function" "vishwesh_lambda" {                                             # → Creates an AWS Lambda function.
  function_name = "vishwesh-lambda"                                                            # → Sets the Lambda function name.
  runtime       = "python3.8"                                                                  # → Uses Python 3.8 as the runtime.
  role          = aws_iam_role.lambda_exec_role.arn                                            # → Assigns the Lambda execution IAM role.
  handler       = "lambda_function.lambda_handler"                                             # → Specifies the entry point for the Lambda function.
  filename      = "lambda.zip"                                                                 # → Uploads the Lambda deployment package.
}

resource "aws_api_gateway_integration" "lambda_integration" {                                  # → Connects API Gateway with Lambda.
  rest_api_id   = aws_api_gateway_rest_api.vishwesh_api.id                                     # → Associates the integration with the API Gateway.
  resource_id   = aws_api_gateway_resource.vishwesh_resource.id                                # → Connects the API resource.
  http_method   = aws_api_gateway_method.vishwesh.http_method                                  # → Uses the configured HTTP method.
  integration_http_method = "POST"                                                             # → Uses POST to invoke the Lambda function.
  type  = "AWS_PROXY"                                                                          # → Enables Lambda Proxy Integration.
  uri   = aws_lambda_function.vishwesh_lambda.invoke_arn                                       # → Points API Gateway to the Lambda function.
}

🔹Jenkinsfile - Deploy Serverless Applications with Lambda
pipeline {                                                                                     # → Starts the Jenkins pipeline.
    agent any                                                                                  # → Runs the pipeline on any available Jenkins agent.
    stages {                                                                                   # → Starts the pipeline stages.
        stage('Build') {                                                                       # → Defines the build stage.
            steps {                                                                            # → Prints the build message.
                echo 'Building Lambda Function...'                                             # → Creates the Lambda deployment package.
                sh 'zip -r lambda.zip lambda/'                                                 # → Creates the Lambda deployment package.
            }
        }
        stage('Deploy to AWS Lambda') {                                                        # → Defines the deployment stage.
            steps {
                script {                                                                       # → Executes shell commands.
                    sh 'aws lambda update-function-code --function-name vishwesh-lambda --zip-file fileb://lambda.zip'   # → Updates the Lambda function code with the new package.   # → Updates the Lambda function code with the new package.
                }
            }
        }
    }
}

🔹 Docker
# Multi-Stage Docker Build
# Stage 1 - Build Stage
FROM maven:3.9-eclipse-temurin-17 AS builder                                                   # → Uses the Maven image as the build stage.
WORKDIR /opt/vishwesh/java-build                                                               # → Sets the working directory for building the application.
COPY . /opt/vishwesh/java-build/                                                               # → Copies the application source code into the build container.
RUN mvn clean package                                                                          # → Builds the Java application and creates the WAR file.

# Stage 2 - Production Stage
FROM tomcat:9                                                                                  # → Uses the Tomcat image as the production stage.
WORKDIR /usr/local/tomcat/webapps                                                              # → Sets the Tomcat deployment directory.
COPY --from=builder /opt/vishwesh/java-build/target/*.war /usr/local/tomcat/webapps            # → Copies the WAR file from the build stage to Tomcat.
EXPOSE 8080                                                                                    # → Exposes port 8080 for the application.
CMD ["catalina.sh", "run"]                                                                     # → Starts the Tomcat server when the container runs.

# Real Paths
/opt/vishwesh/java-build                                                                       # → Directory where the Java application is built.
/usr/local/tomcat/webapps                                                                      # → Directory where Tomcat deploys web applications.


Remember it as like: 
👨‍💻 Developer → Git Push → Jenkins/GitLab CI → Build Lambda Package → Docker Multi-Stage Build → Terraform (Create API Gateway + Lambda) → Ansible (Deploy Lambda) → Kubernetes (Deploy Microservices with Helm) → API Gateway Invokes Lambda → 🚀 Serverless Application Running
