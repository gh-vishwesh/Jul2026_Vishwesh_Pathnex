o Ansible - Setup Nginx with Custom Config
name: Setup Nginx with Custom Config            -----> just title 
hosts: all					-----> do it for all computer
become: yes					-----> become admin first
tasks:
  - name: Install nginx                  	-----> just title
    yum:					-----> task1: aws (the shop)
      name: nginx				-----> means ansible install nginx 
      state: present				-----> if it's already there ansible will not start it.
  -name: Copy custom nginx.conf			-----> task 2 copy	
   copy:	
     src: nginx.conf				-----> the new file
     dest: /etc/nginx/nginx.conf		-----> where it should go
  -name: Start nginx service
   services:					-----> you have installed nginx but it's sleepng
     name:nginx
     state: started 				-----> wake it up
     enabled: yes				-----> if computer reboot nginx stars automatically

o Terraform - Provision EC2 with IAM Role   (Terraform - EC2 with IAM Role)

resource "aws_iam_role" "ec2_role" {           	-----> IAM Role is an identity card for EC2   
  name			= "vishwesh_ec2_role"  	-----> Just a name of the ID card
  assume_role_policy	= jsonencode({		-----> who is allowed to use this ID card?
    Version = "2012-10-17"
    Statement = [
       {
	Action = "sts:AssumeRole"
	Effect = "Allow"
	Principal = {
	  Service = "ec2.amazonaws.com"		-----> means only EC2 instance can use this role, not lambda, not users
	}
      },
    ]
  }}
}

resource "aws_instance" "vishwesh_ec2" {	-----> create one virtual computer
  ami		= "ami-0abcd1234abcd1234"	-----> Think of ami as windows or Linux already installed.
  instance_type = "t2.medium"			-----> size of computer i.e. how powerful it is, here it is medium
  iam_instance_profile = aws_iam_role.ec2_role.name -----> Attach the ID card(IAM Role) to the EC2. Now the EC2 can safely access AWS services without storing passwords.
  tags = {					
    Name = "Vishwesh-EC2"			-----> like writing your name on school bag
  }
}

Remember it as like: Create ID card (IAM Role) -> Allow EC2 to use it -> Create EC2 -> Give server a name

o Kubernetes - ConfigMap and Secret Management
apiVersion: v1
kind: ConfigMap						-----> store non-sensitive Configuration only
metadata:	
  name: vishwesh-config
data:
  app.properties: |					-----> stores configuration value
    key=value

---
apiVersion: v1
kind: Secret						-----> sensitive data like Passwords or API keys are stored in Secrets
metadata:
  name: vishwesh-secret
type: opaque
data:
  password: c6F0a651eHBhc3M=				-----> stores the password

Remember as it like: Recipe -> ConfigMap Password -> Secret

o Shell Script - Archieve Logs
#!/bin/bash
tar -czf /var/log/archived_logs.tar.gz /var/log/*.log	-----> collecting all old files and store them in a archive file, that's what tar does.

Remember for tar: -c : create a new archive
		  -z : Compress it make it smaller
		  -f : Give the archive a file name
/var/log/archived_logs.tar.gz : This is the new compressed file.
and /var/log/*.log : Take every .log file.

Remember it as like: Collect logs -> Compress them -> store in one file


o Docker Java Application
# JavaScript
class Hello {						-----> Create one program.
   public static void main(String[] args){		-----> This is where the program starts. Think of it as the front door.
	System.out.println("Hello Vishwesh"); 		-----> Print something on the screen.
   }
}

# Docker File
FROM openjdk:17						-----> Take a kitchen that already has Java installed.
WORKDIR /opt/vishwesh/java-app				-----> Go inside this folder
COPY Hello.java /opt/vishwesh/java-app/			-----> Bring the java file into the container
RUN javac /opt/vishwesh/java-app/Hello.java		-----> Compile the Java source code into bytecode. This creates a .class file.
CMD ["java", "-cp", "/opt/vishwesh/java-app", "Hello"] 	-----> Run the compiled program.


Remember it as like: Take java -> Go to folder -> copy file -> compile -> run
